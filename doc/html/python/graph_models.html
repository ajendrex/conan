API Reference Guide for Conan's Python Bindings

graph_models module

Graph generator functions

  • Random graphs
    • generate_random_graph (num_vertices, num_edges)
      • This function is just a wrapper for the Boost Graph Library function generate_random_graph.
      • num_vertices: Number of vertices of the graph to be created (integer)
      • num_edges: Number of edges (integer)

      Effects
      Returns a graph with num_vertices vertices and num_edges edges. Source and target vertices of each edge are randomly choosen.
      Preconditions
      num_vertices > 0 and num_edges > 0
      Complexity
      O(2 * V * E)

    • generate_erdos_renyi_graph (num_vertices, p)
      • Generates a graph according to the Erdos-Renyi model for random graphs.
      • num_vertices: Number of vertices (integer)
      • p: Probability for any two vertices to be connected (float)

      Effects
      Returns a graph with num_vertices vertices. The number of edges depend upon the probability p.
      Preconditions
      num_vertices > 0 and 0 ≤ p ≤ 1
      Complexity
      O(V²)

  • Canonical graphs
    • generate_star_graph (num_vertices, num_centers)
      • The star graph is a graph in which all vertices are connected to normally one vertex of the graph. This vertex is called the center of the graph.
      • num_vertices: Number of vertices (integer)
      • num_centers: Number of centers of the star (integer)

      Effects
      Returns a star graph.
      Preconditions
      num_vertices > 0 and num_centers > 0
      Complexity
      O(V * num_centers)

    • generate_regular_lattice (num_vertices, degree)
      • num_vertices: Number of vertices (integer)
      • degree: Number of edges incident to each vertex (integer)

      Effects
      Returns a graph in which each vertex is connected to the closest degree vertices.
      Preconditions
      num_vertices > 0 and degree must be even
      Complexity
      O(V * degree)

    • generate_complete_graph (num_vertices)
      • In a complete graph, all vertices are connected to each other.
      • num_vertices: Number of vertices (integer)

      Effects
      Returns a complete graph.
      Preconditions
      num_vertices > 0
      Complexity
      O(V²)

  • Modern graph models
    • generate_small_world_network (num_vertices, degree, percent_replaced_edges)
      • num_vertices: Number of vertices (integer)
      • degree: The degree which will be used to create the base regular lattice (integer)
      • percent_replaced_edges: Percentage of the total number of edges to be replaced by random edges (float)

      Effects
      Returns a graph that was created based on the Watts-Strogatz model for small-world networks.
      Preconditions
      num_vertices > 0, degree must be even and 0 ≤ percent_replaced_edges ≤ 1
      Complexity
      O()

    • generate_Demetrius_Manke_network (num_vertices, edges_by_vertex, T)
      • num_vertices: Number of vertices (integer)
      • edges_by_vertex: Number of edges to be added by each new vertex added to the graph (integer)
      • T: This parameter determines how strictly the maximal entropic principle is enforced (float)

      Effects
      Returns a graph that was created based on the Demetrius-Manke model for network's evolution.
      Preconditions
      num_vertices > 0 and edges_by_vertex > 0
      Complexity
      O()

      Related function:
      • add_vertex_as_Demetrius_Manke (g, edges_by_vertex, T)
        • g: An undirected_graph
        • edges_by_vertex: Number of edges to be added by each new vertex added to the graph (integer)
        • T: This parameter determines how strictly the maximal entropic principle is enforced (float)

        Effects
        Add a vertex to g according to the network's evolution model of L. Demetrius and T. Manke.
        Preconditions
        edges_by_vertex > 0
        Complexity
        O()

    • generate_scale_free_network (num_vertices, edges_by_vertex)
      • num_vertices: Number of vertices (integer)
      • edges_by_vertex: Number of edges to be added by each new vertex added to the graph (integer)

      Effects
      Returns a graph that was created based on the Barabasi-Albert model of Prefential Attachment.
      Preconditions
      num_vertices > 0 and edges_by_vertex > 0
      Complexity
      O()

      Related function:
      • expand_scale_free_network (g, num_vertices, edges_by_vertex)
        • g: An undirected_graph
        • num_vertices: Number of vertices to be added to g (integer)
        • edges_by_vertex: Number of edges to be added by each new vertex added to the graph (integer)

        Effects
        Add num_vertices vertices to g according to the Prefential Attachment model. Each vertex added connects to the existing graph with edges_by_vertex edges.
        Preconditions
        num_vertices > 0 and edges_by_vertex > 0
        Complexity
        O()
